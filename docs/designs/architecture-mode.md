## Vision

### 10x Check
Ten times more ambitious: VibeLearn becomes the standard tool for measuring and training architecture judgment in the AI-coding era. Every developer using Claude Code, Cursor, or Copilot has a VibeLearn score. "I scored 81 on architecture ownership this month" is a real thing developers say. Hiring managers ask for it. The VS Code Marketplace has it at 50k installs. The product owns the category: "AI-coding literacy."

### Selective Expansion (cherry-pick results)

1. **Codebase onboarding mode** — `vibelearn.explainArchitecture` surfaces "5 design decisions that define this system" for any codebase
2. **"Road not taken" counterfactuals** — detector surfaces the alternative path, not just the chosen one

## Key Interface

```ts
interface ArchitecturalDecision {
  patternType: string;        // e.g. "observer", "dependency-injection"
  decisionName: string;       // "Pub/Sub event bus via EventEmitter"
  tradeoffs: string;          // why this was chosen + when you'd choose differently (canonical source)
  counterfactual: string;     // the alternative path and why it wasn't chosen (new field)
  confidence: number;         // 0–1; only surface if >= 0.8
}
```

## Storage Strategy
- **`knowledge.json`** (existing) — stores `ArchitecturalDecision` entries under `concepts[patternType]`, same SM-2 structure
- **Weekly digest** — local markdown file only, generated on demand via `vibelearn.weeklyDigest`, no new storage format

## Implementation Sequence

Sequence references: B = Implicit Decision Detector, A = Architecture Debrief Layer, C = Ownership Dashboard. Cherry-picks woven in where they fit naturally.

**Phase 1 — Wedge (B + cherry-picks 3 & 5):**
1. `architecture_check` to `InterventionType` + `ArchitecturalDecision` interface in `src/types.ts`
2. `detectArchitecturalDecision()` in `interventionEngine.ts` — pre-pass prompt with counterfactual field; few-shot examples; confidence >= 0.8 threshold
3. Diff-triggered detection in `extension.ts` — debounce 5s, skip diffs < 10 gross lines, 1-queue-max
4. `vibelearn.explainArchitecture` command — reads workspace file structure via `getFileStructure()` (already defined as optional on `SessionAdapter` in `src/types.ts:77`), sends to Claude for "5 defining design decisions" walkthrough, no active session required
- **Phase 1 success signals:** detector fires 1-3x per non-trivial session; dismiss rate < 50%; one developer reports understanding a pattern they didn't before

**Phase 2 — Debrief Layer (A):**
- Extend `generateDebrief()` to read stored `ArchitecturalDecision` structs and surface a "Design Decisions" summary section
- Add "I understand this / I need to dig into this" self-assessment per decision
- **Phase 2 success signal:** "Design Decisions" section engagement > concept check engagement (measured via analytics.jsonl)

**Phase 3 — Ownership Dashboard + Weekly Digest (C):**
- New sidebar view `src/ui/views/architecture.ts`: patterns grouped by category, confidence per pattern
- `vibelearn.weeklyDigest` command: generates local markdown file, clean enough to paste into Slack/Discord
- **Phase 3 success signal:** at least one developer shares the weekly digest publicly

## Eng Review Decisions (2026-03-31)

Resolved during `/plan-eng-review`:

| Decision | Resolution |
|---|---|
| File watching for diff detection | Wire via `vscode.workspace.createFileSystemWatcher` in `extension.ts`, not adapter layer |
| `vibelearn.explainArchitecture` command | Extend existing `vibelearn.explainCodebase` with updated prompt — no new command |
| Async evaluation in `onAnswer()` | Inline async in existing handler, detect `architecture_check` type, use `isGenerating=true` + `panel.showLoading()` |
| ArchitecturalDecision threading | Add `archDecision?: ArchitecturalDecision` to `Intervention` interface — single variable, no sync risk |
| `isDetecting` guard | Separate from `isGenerating` — detection and quiz generation are independent |
| Diff fallback for detection | Skip `HEAD~5` fallback in detection path — only fire on uncommitted changes |
